# Contributing to treemux

Thanks for your interest in contributing!

## Reporting Bugs

- Check existing issues first to avoid duplicates
- Include your OS, tmux version, fzf version, and zsh version
- Provide steps to reproduce the issue

## Feature Requests

- Open an issue describing the feature and why it would be useful
- Be open to discussion about implementation approaches

## Pull Requests

1. Fork the repo and create your branch from `main`
2. Test your changes thoroughly
3. Update documentation if needed
4. Submit a PR with a clear description of the change

## Code Style

- Follow existing code patterns in the script
- Keep it simple - avoid over-engineering
- Use zsh idioms where appropriate

## Development

```bash
# Clone
git clone https://github.com/nicobailon/treemux.git
cd treemux

# Test locally
./treemux

# Check syntax
zsh -n treemux
```

## Questions?

Open an issue - happy to help!
